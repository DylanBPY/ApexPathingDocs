    ```java
    @Override
    public BaseDrivetrainConfig<?> drivetrainConfig() {
    ```

    ```java
    public BaseDrivetrainConfig<?> drivetrainConfig() {
        return new CoaxialSwerve.Config()

                .setBackLeftMotor(new MotorFactory("backLeftMotor"))

                .setWheelbase(Dist.fromMm(0))
                .setOffsetAngle(Angle.fromDeg(0))
                .setSteeringCoefficients(new PDSController.PDSCoefficients());
    }
    ```
    ### Motors
    Replace the names of the motors with the names of your drivetrain motors in the hardware configuration.
    If your motors are reversed, call `.reverse()` for that motor. See more about the [MotorFactory](/docs/classes/motor-factory/) class.
    You can use the `MotorDebugger` OpMode in the [OpModes](/docs/opmodes/) page to get your motor positions and directions correct.

    ### Servos
    Replace the names of the servos with the names of your drivetrain servos in the hardware configuration.

    ### Encoders
    Replace the names of the encoders with the names of your drivetrain encoders in the hardware configuration.

    ### Track width and wheelbase
    Set the track width (distance from left wheel center to right wheel center) and wheelbase 
    (distance from front wheel center to back wheel center) of your robot as a [Dist](/docs/classes/dist/).

    ### Offset angle and steering coefficients
    Use the `SwerveTuner` OpMode in the [OpModes](/docs/opmodes/) page to determine the offset angle
    and PDS steering coefficients of your pods. The offset angle is the static angle offset from 0 
    degrees for your wheels to be facing forwards. It is defined as an [Angle](/docs/classes/angle/).
    The steering coefficients are used to correct positional errors in the steering of your pods.

    You can use the `MotorDebugger` OpMode in the [OpModes](/docs/opmodes/) page to get your motor positions and directions correct.

    ### Actuators
    You can add or remove `.addActuator()` calls depending on how many actuators your drivetrain has. 
    Replace the names of the actuators with the names of your drivetrain actuators in the hardware configuration. 
    The second and third parameters are the `tractionPosition` and `holonomicPosition` of the actuator respectively.

    ### Initial state (optional)
    You can set the initial state of the actuators with `.setInitialState(DualActuated.DriveState initialState)`.
    The two possible states are `DualActuated.DriveState.TRACTION` and `DualActuated.DriveState.HOLONOMIC`.
    The default is `HOLONOMIC`.

## Global Drivetrain Constants
Some constants are common to all drivetrains. All of these are optional and have default values which
are usually sufficient for most robots. You can override them in your drivetrain configuration if you want to customize them.

- `.setRobotCentric(boolean robotCentric)`
    - If true, `follower.teleOpDrive()` will use robot-centric controls, otherwise it will use field-centric controls.
    - Default is true (robot centric)
    - You can swap between robot-centric and field-centric on the fly with
    `follower.setRobotCentric(boolean robotCentric)` in your OpMode.
- `.setMaxPower(double maxPower)`
    - Sets the maximum power that can be applied to drivetrain motors.
    - Default is 1.0 (100% power)
- `setLinearVelocityLimit(Dist linearVelocityLimit)`
    - Sets the maximum linear velocity of the robot as a [Dist](/docs/classes/dist/) in units per second.
    - Default is 0 (disabled, no limit)
- `setAngularVelocityLimit(Angle angularVelocityLimit)`
    - Sets the maximum angular velocity of the robot as an [Angle](/docs/classes/angle/) in units per second.
    - Default is 0 (disabled, no limit)
- `setLinearAccelerationLimit(Dist linearAccelerationLimit)`
    - Sets the maximum linear acceleration of the robot as a [Dist](/docs/classes/dist/) in units per second squared.
    - Default is 0 (disabled, no limit)
- `setAngularAccelerationLimit(Angle angularAccelerationLimit)`
    - Sets the maximum angular acceleration of the robot as an [Angle](/docs/classes/angle/) in units per second squared.
    - Default is 0 (disabled, no limit)
